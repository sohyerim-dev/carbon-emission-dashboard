# GreenWave Carbon

> 기업의 탄소 배출 현황을 시각화하고, 예상 탄소 비용 및 감축 시나리오를 분석할 수 있는 탄소 배출 대시보드

## 프로젝트 소개

GreenWave Carbon은 기업 임원 및 매니저가 탄소 배출 현황을 빠르게 파악하고, 향후 발생 가능한 탄소 비용과 감축 전략을 계획할 수 있도록 설계한 웹 기반 탄소 배출 대시보드입니다.

단순히 배출량 데이터를 나열하는 것이 아니라 아래 질문에 답할 수 있도록 정보 구조를 설계했습니다.

1. 현재 우리 기업의 총 탄소 배출 규모는 어느 정도인가?
2. 어떤 배출원(Source)이 가장 큰 영향을 주고 있는가?
3. 현재 기준 예상 탄소 비용은 얼마인가?
4. 감축 목표를 적용했을 때 비용 절감 효과는 어느 정도인가?

이를 위해 대시보드의 흐름을 아래 순서로 구성했습니다.

```text
현황 파악 → 원인 분석 → 비용 영향 → 공시 확인
```

# 주요 기능

## 대시보드

- 총 배출량 / 예상 탄소세 / 기간 대비 변화율 요약 카드
- 기간 필터 (이번 달 / 최근 3개월 / 최근 1년 / 전체 / 연도별)
- Scope 및 Source 기반 월별 배출 추이 시각화
- 기업별 배출량 비교
- 배출원(Source) 비중 분석 (알파벳 식별자 + 범례)
- 최근 공시 목록 및 수정·삭제 관리

## 탄소세 시뮬레이터 (대시보드 인라인)

- 탄소 시장 프리셋 선택 (K-ETS / EU ETS / California CaT / 직접 입력)
- 감축 목표 슬라이더 기반 예상 절감 비용 계산
- 상세 시뮬레이션 페이지 연결 CTA

## 공시 관리

- 공시 수정 (모달 기반) / 삭제 (즉시 반영)
- 저장 중 / 완료 / 오류 / 재시도 상태 처리
- optimistic update 및 rollback 처리

# 가정 및 설계 결정 (Assumptions & Design Decisions)

## 데이터 모델 확장

과제 기본 데이터 모델에 아래 항목을 추가했습니다.

### `Country.defaultCarbonPrice`

국가별 탄소 가격 프리셋 제공을 위해 추가했습니다.

실제 탄소 시장은 국가 및 규제 체계에 따라 가격 차이가 존재합니다.

예:

- K-ETS ($6.45/tCO₂e)
- EU ETS ($70.37/tCO₂e)
- California CaT ($29.27/tCO₂e)

출처: World Bank Carbon Pricing Dashboard 2025

이를 통해 탄소세 시뮬레이터에서 현실적인 비용 계산이 가능하도록 했습니다.

---

### `Company.industry`

산업군별 탄소 배출 특성이 다르기 때문에 추가했습니다.

예:

- IT 기업 → Scope 2 중심
- 물류 기업 → Scope 1 중심
- 유통 기업 → Scope 3 중심

또한 동종업계 벤치마크 비교 기능을 구현하기 위해 산업 분류가 필요했습니다.

---

### `Post.category`

공시 데이터를 아래 유형으로 구분하기 위해 추가했습니다.

- announcement
- report
- news

경영진이 공시와 일반 뉴스를 구분해 탐색할 수 있도록 설계했습니다.

# 대시보드 설계 의도

이 프로젝트는 단순히 배출량을 나열하는 화면이 아니라,
경영진이 빠르게 의사결정을 내릴 수 있는 구조를 목표로 설계했습니다.

### 정보 구조

```text
요약 → 추세 분석 → 원인 분석 → 비용 영향 → 공시 확인
```

### 배출원(Source) 중심 분석

Scope 1/2/3 비중 대신 실제 배출원(Source) 기준 도넛 차트를 구성했습니다.

이유:

- 고정 연소 (공장 보일러)
- 이동 연소 (차량·선박)
- 구매 전력
- 공급망

등 실제 배출 활동 기준으로 보여주는 것이
경영진 입장에서 "무엇을 줄여야 하는가"를 더 직관적으로 파악할 수 있다고 판단했기 때문입니다.

# 탄소세 시뮬레이터 설계

대시보드에는 간소화된 시뮬레이터를 제공하고,
상세 기능은 별도 페이지(`/simulator`)로 분리했습니다.

### Dashboard 인라인 위젯

- 탄소 시장 선택 (K-ETS / EU ETS / CA CaT / 직접 입력)
- 예상 비용 계산
- 감축 목표 슬라이더
- 예상 절감액 확인

메인 대시보드는 빠른 의사결정,
상세 페이지는 심층 분석에 집중하도록 역할을 분리했습니다.

# 공시 수정 UX 결정

공시 수정은 인라인 편집 대신 모달 기반으로 구현했습니다.

인라인 편집은 테이블 레이아웃 변경과 상태 복잡도가 증가하는 반면,
모달은 수정 상태와 로딩/에러 처리를 독립적으로 관리할 수 있어
fake backend의 실패/재시도 시나리오를 더 명확하게 표현할 수 있다고 판단했습니다.

- 수정: 모달 내 인라인 에러 메시지 + 재시도 버튼
- 삭제: 즉시 반영, 실패 시 rollback 처리

# 기간 필터 설계

헤더 영역에 상대 기간 기반 필터를 배치했습니다.

- 이번 달 / 최근 3개월 / 최근 1년 / 전체 (버튼 그룹)
- 연도 드롭다운 (2024 / 2025)

상대 기간과 연도 드롭다운을 분리한 이유는,
데이터 연도가 추가되더라도 버튼 수가 늘어나지 않도록 확장성을 유지하기 위함입니다.

# 데이터 상태 처리

fake backend 요구사항을 반영하기 위해
로딩 / 실패 / 재시도 흐름을 명확하게 분리했습니다.

## 처리 전략

- Skeleton UI 기반 로딩 상태
- 모달 내 인라인 에러 메시지 + 재시도
- optimistic update (삭제 즉시 반영)
- rollback 처리 (삭제/수정 실패 시 복구)
- SWR 기반 stale-while-revalidate 전략

또한 사이드바 하단에 API 상태를 노출해
현재 데이터 동기화 상태를 사용자에게 지속적으로 제공합니다.

## 계절 패턴 반영

실제 산업 환경처럼 계절에 따른 배출량 변동을 반영했습니다.

# AI 활용 방식

이번 과제에서는 AI를 단순 코드 생성 도구보다는
설계 리뷰 및 아이디어 탐색 보조 역할로 활용했습니다.

## 활용 영역

- 대시보드 정보 구조(IA) 탐색
- UX 트레이드오프 검토
- README 문서 구조화
- 데이터 시나리오 아이디어 확장

## 사용 예시

### 정보 구조 탐색

Prompt:

```text
기업 임원용 탄소 배출 대시보드라면
어떤 정보 구조가 의사결정에 적합할지 제안해줘.
```

### UX 트레이드오프 검토

Prompt:

```text
공시 수정 기능에서
인라인 편집과 모달 편집의 장단점을 비교해줘.
fake backend 실패 처리까지 고려해서 추천해줘.
```

## 의사결정 방식

AI 제안을 그대로 적용하지 않고,
과제 요구사항과 사용자 흐름에 맞는지 직접 검토 후 선택했습니다.

예를 들어 초기에는 기업 목록을 사이드바에 배치하는 구조를 고려했지만,
관리 툴 느낌이 강하다고 판단해 제거했습니다.
